# Фаза 5 — Деплой

## Статус
- Статус: ✅
- Дата/время: 2026-04-07 07:51:50 +05:00

## Цель фазы
- Развернуть рабочий экземпляр `Voice Inbox` на сервере, поднять systemd-сервис и проверить базовую доступность бота и веб-панели.

## Что сделано
- Код клонирован на сервер в `/root/voice-inbox`.
- На сервере создано виртуальное окружение `/root/voice-inbox/.venv`.
- Установлены Python-зависимости из `requirements.txt`.
- Сгенерирован `WEB_PASSWORD`.
- Создан серверный `/root/voice-inbox/.env`.
- Создан unit `/etc/systemd/system/voice-inbox.service`.
- Выполнены `systemctl daemon-reload`, `systemctl enable --now voice-inbox`.
- Проверена внешняя доступность веб-панели.

## Проблемы и отклонения
- Существенных блокеров в этой фазе не возникло.
- Так как деплой выполнялся с Windows на Linux, при создании серверного `.env` и systemd unit нужно было избежать CRLF/BOM-проблем.

## Как решено
- Для серверных текстовых файлов использованы временные локальные файлы с UTF-8 без BOM и LF-окончаниями строк.
- В серверный `.env` дополнительно добавлен `PUBLIC_BASE_URL`, чтобы бот мог строить абсолютные ссылки на веб-панель.
- Проверка `ufw` отработала штатно: firewall был не активен, отдельное правило не потребовалось.

## Проверки
- `systemctl is-active voice-inbox` → `active`
- В `journalctl -u voice-inbox --no-pager -n 40` подтверждены:
  - старт Waitress на `0.0.0.0:8080`
  - старт Telegram application
  - `getMe`, `deleteWebhook`, `getUpdates`
  - успешный веб-логин
- `GET http://94.103.178.210:8080/login` → `200`
- `GET http://94.103.178.210:8080/` без cookie → `302` на `/login`
- Логин с корректным паролем открывает ленту (`INDEX_OK`)

## Артефакты и изменения
- Созданы/обновлены серверные артефакты:
  - `/root/voice-inbox`
  - `/root/voice-inbox/.venv`
  - `/root/voice-inbox/.env`
  - `/etc/systemd/system/voice-inbox.service`
  - systemd-сервис `voice-inbox`
- Сгенерирован `WEB_PASSWORD`; raw значение в отчёт не записывалось, оно было сообщено пользователю отдельно.
